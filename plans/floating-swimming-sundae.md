# 注射と針の互換性検索 — 実行計画書

> **ステータス**: 設計完了・実装待ち（2026-04-10作成）
> **次のセッションで実装開始すること**

---

## 1. 概要

医療従事者が現場で注射器と針の互換性をすばやく調べるWebアプリ。
バックエンド不要の純粋HTML+JS+CSS構成。`python -m http.server` で起動。

| 項目 | 決定内容 |

## 4. 実装手順（この順番で作る）

### Step 1: `data/compatibility.csv`（サンプルデータ）

UTF-8で保存。RFC 4180準拠（ダブルクォート含むフィールドはクォートで囲む）。

```csv
syringe_maker,syringe_model,needle_maker,needle_model,notes
テルモ,"1mL テルモシリンジ",テルモ,"27G 1/2""",
テルモ,"1mL テルモシリンジ",テルモ,"25G 5/8""",若干滲漏あり
テルモ,"3mL テルモシリンジ",テルモ,"23G 1""",
テルモ,"3mL テルモシリンジ",テルモ,"27G 1/2""",
ケアフィールド,"1mL シリンジ",テルモ,"27G 1/2""",
ケアフィールド,"1mL シリンジ",ニプロ,"25G 5/8""",
ニプロ,"2.5mL シリンジ",ニプロ,"23G 1 1/4""",
ニプロ,"2.5mL シリンジ",テルモ,"22G 1 1/4""",
```

### Step 2: `style.css`

- カラー: 背景白 `#FFFFFF`、アクセント `#0077B6`、カード背景 `#F8FAFE`
- カード: 角丸 `8px`、ドロップシャドウ、メーカー名小さめ、製品名大きめ、備考グレー小文字
- レスポンシブ: flexbox/gridでPC・タブレット・スマホ対応
- タブ: アクティブタブにアクセントカラーの下線
- オートコンプリート: position absoluteのドロップダウン、選択中候補にハイライト背景色
- 状態メッセージ: 読み込み中（グレー）、エラー（赤系）、結果なし（グレー）

### Step 3: `app.js`

**依存**: PapaParse（index.htmlでCDN読み込み）

実装する機能：

1. **CSV読み込み・パース**
   - `fetch('data/compatibility.csv')` → テキスト取得 → `Papa.parse(text, { header: true, skipEmptyLines: true })`
   - 失敗時はエラーメッセージをUIに表示

2. **データ保持**
   - パース結果をグローバル配列 `compatibilityData` に格納
   - 注射器ユニークリスト: `{ maker, model }` の重複排除済みリスト
   - 針ユニークリスト: `{ maker, model }` の重複排除済みリスト

3. **ひらがな→カタカナ変換関数**
   - `toKatakana(str)`: ひらがなをカタカナに変換（charCode + 96）
   - 検索時は入力値・候補値の両方をカタカナに変換してから比較

4. **オートコンプリート**
   - `input` イベントで発火
   - タブに応じて注射器リスト or 針リストから `メーカー + " " + 製品名` の部分一致検索
   - 大文字小文字・ひらがなカタカナ区別なし
   - 最大10件表示
   - 各候補は表示用テキストと `{ maker, model }` を両方持つ（data属性 or クロージャ）
   - キーボード操作: ↑↓で候補移動、Enterで選択、Escで閉じる
   - 候補クリックでも選択可能

5. **検索実行**
   - 候補選択時に `{ maker, model }` で `compatibilityData` をフィルタ
   - タブ1: `syringe_maker === maker && syringe_model === model` → 針側のデータを返す
   - タブ2: `needle_maker === maker && needle_model === model` → 注射器側のデータを返す

6. **結果カード生成**
   - 各結果を `<div class="card">` で生成
   - メーカー名、製品名、備考（空なら非表示）
   - 0件時:「該当する製品が見つかりませんでした」

7. **タブ切り替え**
   - タブクリック時にアクティブ切り替え＋入力クリア＋結果クリア

### Step 4: `index.html`

```text
構造:
<head>

    <button class="tab">針から注射器を探す</button>
  </div>
  <div class="search-container">
    <input type="text" placeholder="製品名やメーカー名を入力...">
    <div class="autocomplete-list"><!-- JS生成 --></div>
  </div>
  <div class="status-message"><!-- 読み込み中/エラー/結果なし --></div>
  <div class="results"><!-- カードがJS生成される --></div>
  <script src="app.js"></script>
</body>
```

## 5. 設計上の注意点（レビューで指摘済み）

- **CSVパース**: 自前split(',')は禁止。PapaParseを使う（ダブルクォート・カンマ対応）
- **重複排除**: オートコンプリート候補はmaker+modelの組で重複排除する
- **maker/model分離**: 表示用文字列と内部オブジェクトを分離して保持する
- **エラー状態**: 読み込み中・失敗・結果0件の3状態をUI表示する
- **UTF-8**: CSVは必ずUTF-8で保存（Shift-JISだと文字化け）
- **キーボード操作**: ↑↓Enter Escに対応する
