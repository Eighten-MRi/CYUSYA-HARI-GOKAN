# 別表第九 在宅自己注射対象薬剤 — 商品名レベルまでの整理・統合プラン

## Context

薬局で処方箋に基づき患者に渡せる注射薬の一覧を調査。ローカルPDF（`data/001440805.pdf`、中医協 令和7年3月12日付）に**別表第九**の完全リスト（79カテゴリ＋追加案3）が含まれていることを確認した。

**問題**: 別表第九は「インスリン製剤」「エタネルセプト製剤」のような**一般名（薬効カテゴリ）レベル**。実際に薬局で扱う**商品名**（ノボラピッド注フレックスタッチ等）まで落とし込む必要がある。1カテゴリが数十商品に展開されるため、**処方頻度が高い順**に優先して調査する。

## 実装ステップ

### Step 0: 情報源の保存（最優先）
実装に入る前に、調査で得た情報を2つのファイルに保存する。

#### 0-A: 参照ドキュメント `data/sources.md`
- 調査日時（2026年4月12日）
- 使用した情報源URL一覧と各ソースの概要
- 別表第九の79カテゴリ＋追加3品目の完全リスト（一般名）
- PDF出典情報（`001440805.pdf`の文書タイトル・発行元・日付）
- データの信頼度・注意事項

#### 0-B: 商品名データファイル `data/brand_names_raw.csv`
- 今回のWeb調査で取得できた商品名データをそのまま構造化保存
- Priority 1〜6の主要カテゴリ分（インスリン約40品目、GLP-1約30品目、生物学的製剤約15品目、皮膚科約10品目、骨粗鬆症3品目、その他約10品目）
- スキーマ: `category_id,generic_name,brand_name,maker,device_type,device_name,dose,frequency,source_url`
- 後続のStep 1でこのファイルを元にc101_medications.csvへ加工

### Step 1: 薬剤マスターCSVの作成
`data/c101_medications.csv` を新規作成。**商品名レベル**で記録する。

**スキーマ:**
```csv
id,category_id,generic_name,brand_name,maker,device_type,device_name,dose,administration_route,frequency,indication,self_injection,source,last_verified
```

- `category_id`: 別表第九の番号（1〜82）
- `device_type`: ペン型 / プレフィルドシリンジ / プレフィルドペン / オートインジェクター / アテオス / 注射器+バイアル / 注入ポンプ / SD型
- `self_injection`: 可 / 可（条件付き） / 不可
- まずPriority 1〜6の主要品目（約80〜100商品）を入力
- 残りのカテゴリは後続セッションで順次追加

### Step 2: 既存compatibility.csvとの連携
- `compatibility.csv` のプレフィルドカテゴリ（ヒュミラ/エンブレル/シムジア）に `category_id` フィールドを追加
- `c101_medications.csv` の `category_id` と紐付けて、薬剤→デバイス→対応する針の検索を実現

### Step 3: UIへの統合
- 新タブ「薬剤から検索」を追加
- 薬剤名のオートコンプリート検索 → 該当デバイス表示 → 互換性のある針情報へのリンク

## 検証方法
1. CSVのPriority 1〜6で80〜100行の商品データが入っていることを確認
2. 既存compatibility.csvの3品目（ヒュミラ/エンブレル/シムジア）とcategory_idでマッチすることを確認
3. `python -m http.server 8000` → 薬剤名検索で商品名候補が表示されることを確認
4. 薬剤選択→デバイス情報→対応する針の情報へのナビゲーションが動作すること

## 注意事項
- デバイスタイプは調査時点の情報。新剤形の追加や販売中止あり → `last_verified` で管理
- バイオシミラー（BS）は先発品とデバイスが異なる場合がある → 個別に確認要
- リベルサス（セマグルチド内服）等の非注射製剤は対象外
- この文書の「案」部分（3品目追加）は中医協承認後に正式反映
