# CSVデータベース拡充プラン

## Context

現在 `data/compatibility.csv` にサンプル8行・5列しかない。メーカーカタログやPMDA添付文書を手動で参照しながら、実用的なデータセットに育てたい。同時にCSVスキーマも拡張し、接続規格・容量・ゲージなど検索・表示に有用な列を追加する。アプリ側もそれに合わせて結果カードの表示を強化する。

## Phase 1: CSVスキーマ拡張

**ファイル**: `data/compatibility.csv`

現行5列 → 11列に拡張:

```csv
syringe_maker,syringe_model,syringe_volume_ml,syringe_tip_type,needle_maker,needle_model,needle_gauge,needle_length,needle_connection,compatibility,notes
```

| 列名 | 説明 | 例 |
|------|------|-----|
| `syringe_maker` | 注射器メーカー（既存） | テルモ |
| `syringe_model` | 注射器名（既存） | テルモシリンジ SS-01T |
| `syringe_volume_ml` | **新規** 容量(mL) | 1 |
| `syringe_tip_type` | **新規** 先端規格 | ルアースリップ / ルアーロック |
| `needle_maker` | 針メーカー（既存） | テルモ |
| `needle_model` | 針の名前（既存） | テルモ注射針 NN-2725S |
| `needle_gauge` | **新規** ゲージ | 27G |
| `needle_length` | **新規** 長さ | 1/2" |
| `needle_connection` | **新規** 接続方式 | ルアースリップ / ルアーロック |
| `compatibility` | **新規** 互換状態 | 確認済 / 規格適合 |
| `notes` | 備考（既存） | 若干滲漏あり |

既存の8行のデータを新スキーマに移行し、新列を埋める。

## Phase 2: サンプルデータ投入

既存8行のデータを新スキーマに変換した上で、主要メーカー（テルモ・ニプロ・ケアフィールド）の一般的な製品の組み合わせを追加し、約30-50行程度のデータセットにする。

データ入力の方針:
- 接続規格ベースの互換性ルールに基づく（ルアーロック注射器→全針OK、ルアースリップ注射器→スリップ針のみ）
- `compatibility` は手動確認済みなら「確認済」、規格から推定なら「規格適合」
- `notes` に注意事項があれば記載

## Phase 3: app.js 更新

**ファイル**: `app.js`

### 3-1. 結果カードの表示拡張
`performSearch()` 関数を修正して、新しい列（容量・ゲージ・長さ・接続規格・互換状態）を結果カードに表示する。

現状:
```
[メーカー名]
[モデル名]
[備考]
```

変更後:
```
[メーカー名]
[モデル名]
[容量/ゲージ] [接続規格]    ← 新規
[互換状態]                   ← 新規
[備考]
```

### 3-2. オートコンプリートの表示改善
`showAutocomplete()` で候補表示にも容量やゲージ情報を含めて、同名製品を区別しやすくする。

### 3-3. buildUniqueLists の拡張
新列（volume, tip_type, gauge, length, connection）もユニークリストに含めて、検索候補の情報量を増やす。

## Phase 4: style.css 更新

**ファイル**: `style.css`

結果カードに追加した新情報（接続規格バッジ、容量/ゲージ表示）のスタイルを追加。

## 修正対象ファイル一覧

| ファイル | 変更内容 |
|---------|---------|
| `data/compatibility.csv` | スキーマ拡張 + 既存データ移行 + サンプルデータ追加 |
| `app.js` | 結果カード表示拡張、オートコンプリート改善、ユニークリスト拡張 |
| `style.css` | 新しいカード要素のスタイル追加 |

`index.html` は変更不要（動的にJS側で生成しているため）。

## 検証方法

1. ローカルサーバーで `index.html` を開く
2. 「注射器から針を探す」タブで「テルモ」と入力 → 候補に容量・規格情報が表示されるか
3. 候補を選択 → 結果カードに新列（ゲージ・長さ・接続規格・互換状態）が表示されるか
4. 「針から注射器を探す」タブでも同様に動作するか
5. 備考欄のある行・ない行の両方で表示が崩れないか
